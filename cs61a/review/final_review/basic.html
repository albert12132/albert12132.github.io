  • 'explanation': """For the solution using next,
  • let's first look at the init. Since iter1
  • and iter2 are defined to be iterables, not
  • iterators, we need to first call iter on them
  • to extract their iterators (this is so we can call
  • next later on). We also extract the first elements
  • in the iterators to get self.elem1 and
  • self.elem2. This will become useful later on.

  • Recall that the iter method must always return

  • an iterator, and an iterator, by definition, must have a
  • next method. Since we are returning self,
  • this implies our class must have a next method.

  • In next, we compare the current elements of

  • both iterators. If self.comp returns True, this
  • signals that the next element to return will be the element
  • from the first iterator. We save self.elem1 inside a
  • temporary variable, and then overwrite self.elem1 with
  • the next element in self.iter1. We do something
  • similar if self.comp returns False instead.

  • For the version that uses a generator expression, observe

  • that, by using a yield statement, we convert the
  • iter method into a generator function. By their
  • nature, generator functions always return iterators, so this
  • fits the requirement that iter must return an
  • iterator. The logic in this version is very much the same as
  • before. The only difference is we don't need to save instance
  • variables for elem1 and elem2, since the
  • generator function can save state for us."""
