Markdown

Last updated: Jul 24, 2014

Templar uses a specially-built parser to process Markdown. In addition to the original Markdown specification, Templar also includes a subset of Markdown Extra features. This article documents all of the Markdown rules used by Templar.

Metadata

One unconventional feature of Templar's Markdown parser is the metadata syntax, also referred to as variable syntax:

~ title: Example

Some content
<p>Some content</p>

The line that says ~ title: Example is the metadata declaration. This line declares a variable called title whose value is Example. Metadata is not accessible from within a Markdown document, but it is available to Templar's template compiler.

Metadata can be defined anywhere within the Markdown document. All metadata is stripped from the final document.

Headers

Markdown supports two styles of headers: atx headers and Setext headers.

atx headers

Markdown's atx headers use one to six #s:

## Level 2 header
### Trailing #s ignored ###
<h2 id="level-2-header">Level 2 header</h2>
<h3 id="trailing-s-ignored">Trailing #s ignored</h3>

Notice that id attributes are added to the headers. These ids are derived from the contents of the header by lower-casing all letters, hyphenating spaces, and removing punctuation.

Setext headers

Setext headers rely on underscores and can be used to generate level 1 or level 2 headers:

Level 1 header
==============

Level 2 header
--------------
<h1 id="level-1-header">Level 1 header</h1>

<h2 id="level-2-header">Level 2 header</h2>

As with atx headers, Templar derives "slug" id attributes for the headers.

Special Attributes

Markdown Extra defines syntax for supplying certain elements with "special attributes." Templar has implemented this feature only for headers:

Header 1      {#header1.title.stuff}
========

## Header 2   { #header2 }
<h1 id="header1" class="title stuff">Header 1</h1>

<h2 id="header2">Header 2</h2>

The attributes used inside of the { ... } follow CSS class and id syntax.

HTML elements

Any HTML elements are preserved:

This is due <span class="date">12/21</span>
<p>This is due <span class="date">12/21</span></p>

Block level HTML tags will be preserved only if they occur at the start of a line (that is, no preceding characters on the line) and if the blocks are separated from surrounding content by blank lines:

Some stuff

<table>
  <tr>
    <td>entry 1</td>
    <td>entry 2</td>
  </tr>
</table>
<p>Some stuff</p>

<table>
  <tr>
    <td>entry 1</td>
    <td>entry 2</td>
  </tr>
</table>

Block-level elements

Paragraphs

A paragraph is just a block of text. Multiple paragraphs are separated by at least one blank line:

This is paragraph 1. It contains content
that pertains to the first paragraph.

This is paragraph 2. Notice the blank line
above the start of this paragraph.
<p>This is paragraph 1. It contains content
that pertains to the first paragraph.</p>

<p>This is paragraph 2. Notice the blank line
above the start of this paragraph.</p>

To force line breaks (<br/>) instead of a new paragraph, add two spaces at the end of the last line. In the example below, the ~~ represents two spaces:

This paragraph will
contain a line break~~

This is still part of the
same paragraph
<p>This paragraph will
contain a line break<br/>

This is still part of the
the same paragraph</p>

Code blocks

Code blocks are denoted by adding at least four spaces to the front of each line in the code block:

This text is just in a normal paragraph

    def hello(world):
        return world > 2

    print(hello(3))

Back to normal paragraphs
<p>This text is just in a normal paragraph</p>

<pre><code>def hello(world):
    return world &gt; 2

print(hello(3))</code></pre>

<p>Back to normal paragraphs</p>

Notice that whitespace that occurs after the four leading spaces is preserved. In addition, the characters >, <, and & are escaped to HTML entities; this makes it more conveninet to write code blocks.

Blockquotes

Blockquotes are preceded with >. Blockquotes may contain lists, headers, paragraphs, code blocks, and nested blockquotes:

> ### Beginning of a blockquote
> This is an example of a block quote
> 
>     Here is a codeblock
<blockquote><h3 id="beginning-of-a-blockquote">Beginning of a blockquote</h3>

<p>This is an example of a block quote</p>

<pre><code>Here is a codeblock</code></pre></blockquote>

Lists

Unordered lists

Un-ordered lists look like the following:

* List item 1
* List item 2
* List item 3
<ul>
  <li>List item 1</li>
  <li>List item 2</li>
  <li>List item 3</li>
</ul>

The "bullets" can be any of the following characters: *, +, -. These characters can be used interchangeably in a single list.

Ordered Lists

Ordered lists look like the following:

1. List item 1
2. List item 2
3. List item 3
<ol>
  <li>List item 1</li>
  <li>List item 2</li>
  <li>List item 3</li>
</ol>

The numbering of the lists does not matter. As long as list items are of the format \d+\., they will be considered as the next list item.

Nested paragraphs inside lists

If a list item contains multiple paragraphs, each paragraph after the first one must be indented at least two spaces. This differs from original Markdown syntax, which requires 4 spaces.

* This is a list item.
  The list item will have two
  paragraphs.

  This is the second paragraph
  in the list item.
* This is the second list item.
<ol>
  <li><p>This is a list item.
  The list item will have two
  paragraphs.</p>

  <p>This is the second paragraph
  in the list item.</p></li>
  <li>This is the second list item.</li>
</ol>

Nested lists inside lists

Lists can be nested inside of other lists by adding four leading spaces for every level of nesting. The type of list (unordered or ordered) does not have to be the same as the outer list:

1. Ordered list item 1
    * Unordered list item 1
    * Unordered list item 2
        * Inner unordered list item 1
    * Unordered list item 3
2. Ordered list item 2
<ol>
  <li><p>Ordered list item 1</p>

  <ul>
    <li>Unordered list item 1</li>
    <li><p>Unordered list item 2</p>

    <ul>
      <li>Inner unordered list item 1<li>
    </ul></li>
    <li>Unordered list item 3</li>
  </ul></li>
  <li>Ordered list item 2</li>
</ol>

Nested code blocks in lists

To include a code block inside of a list, the code block requires four leading spaces for every list level. For example:

* List item

        Code block with 8 trailing spaces:
        4 spaces for 1 list level
        + 4 spaces for code block

  End list item
    * Nested list item

            Code block with 12 trailing spaces:
            8 spaces for 2 list levels
            + 4 spaces for code block
<ol>
  <li><p>List item</p>

  <pre><code>Code block with 8 trailing spaces:
  4 spaces for 1 list level
  + 4 spaces for code block</code></pre>

  <p>End list item</p>

  <ul>
    <li><p>Nested list item</p>

    <pre><code>Code block with 12 trailing spaces:
    8 spaces for 2 list levels
    + 4 spaces for code block</code></pre></li>
  </ul></li>
</ol>

Nested blockquotes in lists

As with code blocks, blockquotes nested in lists also require four leading spaces per list level:

* List item

    > Block quote
    > with 4 leading spaces

  End list item
    * Nested list item

        > Block quote
        > with 8 leading spaces
<ol>
  <li><p>List item</p>

  <blockquote>Block quote
  with 4 leading spaces</blockquote>

  <p>End list item</p>

  <ul>
    <li><p>Nested list item</p>

    <blockquote>Block quote
    with 8 leading spaces</blockquote></li>
  </ul>
</ol>

Tables

Templar implements Markdown Extra's syntax for pipe-styled tables:

Header 1 | Header 2 | Header 3
:-------:|---------:|:--------
center   | right    | left
<table>
  <tr>
    <th>Header 1</th>
    <th>Header 2</th>
    <th>Header 3</th>
  </tr>
  <tr>
    <td align="center">center</td>
    <td align="right">right</td>
    <td align="left">left</td>
  </tr>
</table>

The colons control which alignment to use for each column:

  • Two colons, one on each side, converts to center alignment
  • One colon on the right converts to right alignment
  • One colon on the left converts to left alignment

Span elements

Bold and italics

Templar uses the same syntax as original Markdown for emphasis:

*one star* and _one underscore_
**two stars** and __two underscores__
***three stars*** and ___three underscores___
<p><em>one star</em> and <em>one underscore</em>
<strong>two stars</strong> and <strong>two underscores</strong>
<strong><em>three stars</em></strong> and <strong><em>three underscores</em></strong></p>

Markdown Extra's flavor of underscores is not used — that is, underscores in the middle of words are still treated as emphasis, not literal underscores:

not_good_example
<p>not<em>good</em>example

Code

Code spans are denoted by backticks:

This is `in code` tag.
Use ``more `s if you need`` a backtick in the tag
<p>This is <code>in code</code> tag
Use <code>more `s if you need</code> a backtick in the tag</p>

At this time, fenced code blocks are not supported.

Links

Hyperlinks

Hyperlinks are denoted as follows:

This is a [hyperlink](http://www.example.com).
This [link](www.test.com "title attr") has a title attribute
<p>This is a <a href="http://www.example.com">hyperlink</a>.
This <a href="www.test.com" title="title attr">link</a> has a title attribute</p>

Images

Images have similar syntax to hyperlinks but are preceded with an exclamation mark:

![image 1](example.png)
![image 2](example.jpg "title attr")
<img src="example.png" alt="image 1"/>
<img src="example.jpg" alt="image 2" title="title attr"/>

References

References can be defined anywhere in the Markdown document and be used for links and images:

This is a [hyperlink][an id].

   [an id]: www.example.com
<p>This is a <a href="www.example.com">hyperlink</a>.</p>

Reference definitions may be indented up to three spaces.

Implicit references can also be made: if the second set of [] is left blank, the content in the first set of [] is used as the reference ID:

This goes to [Google][].

   [google]: www.google.com
<p>This goes to<a href="www.google.com">Google</a>.</p>

Notice that reference IDs are case insensitive.

Footnotes

Markdown Extra defines syntax for footnotes:

This is a footnote.[^id] And this is
another footnote.[^2]

   [^2]: Footnotes!
   [^id]: This is the footnote.
<p>This is a footnote.<sup><a href="#fnref-1">1</a></sup> And this is
another footnote.<sup><a href="#fnref-1">2</a></sup></p>

<hr/>

<div id="footnoes">
</div>
  <ol>
    <li id="fnref-1">This is the footnote.</li>
    <li id="fnref-2">Footnotes!</li>
  </ol>

Miscellaneous

Tabs and Whitespace

All tabs are converted into at most 4 spaces. All trailing whitespace (\s+$) is removed.

Emdashes

Occurences of -- that are converted into em-dashes, as long as they are

  • not part of Setext headers
  • not part of a horizontal rule
  • not in a code block or a code span
  • not in an HTML comment
The code--submitted long ago--ran incorrectly.
<p>The code&mdash;submitted long ago&mdash;ran incorrectly.</p>

Comments

Templar supports Pandoc-style comments:


&lt;!--- Use three leading hyphens to create a
Pandoc comment. These comments are removed
from the final document —&gt;

Pandoc comments are completely removed before any other Markdown processing occurs.
